7 tips for a Node.js padawan

Sep 24th, 2013 Comments

一些我更愿意在开始就知道东西

利用 Node.js 开发是一个非常有趣,和令人满足的过程, 他有3万多个模块可以选择使用,并且所有的模块可以非常容易的集成入现有的应用之中.

无论如何,对于一些刚开始使用Node.js 开发的的人来说, 很容易碰壁,在这个很多的文章中,我会提到在你学习过程中遇到的问题 .

贴士 1: 在开发环境使用 nodemon, 在生产环境使用pm2

当你第一次开发Node.js 应用的时候, 其中一件事情就是一次又一次的运行 file.js 就和揭伤疤一样. 当我第一次开发的node的时候,这个让我感到挫败和痛苦, 尤其是每当我修改很小东西的时候需要 control+c

幸运的是我发现了一个非常棒的工具 Nodemon. 你可以利用以下的命令来安装

Nodemon 是一个值得称赞的工具, 当你全局安装他以后, 可以痛过 nodemon [file].js 来启动你的node.js scripts,它会告诉nodemon来监视你的script和所有的scripts有变化的时候,这是个令人敬畏的Node.js开发方式以及让提高开发速度.

那么,生产环境又如何, 除非你用了 heroku, Nodejitsu 或者其他一些好的 Node.js 平台, 碰巧你用了EC2 或者一些其他的云平台来运行你的Node.js app, 你如何能然保证这是一个始终运行的Node.js app?

这个问题的答案就是 PM2,PM2 是一个类似于Nodemon的工具用于生产环境, 和Nodemon相似的地方在于他会监控你的app每当任何修改或者重新部署的时候,但是有更好的一面, PM2 在遭遇到崩溃的时候,他会正确重启你的app.

PM2的优胜之处在于当你要将app需要多核处理的时候,PM2内部集成的负载均衡可以让你很容易的去指定多少个实例在运行.

-i 参数是为了指定运行多少个实例,在这个例子中 PM2 使用了一个常量 max 来扩展你的app 到你最大的核数,不要忘记Node 平时只会运行在单核!

贴士 3: 轻松调试 Node.js apps

如果你从一个IDE重度集成的语言比如java 或者C# 转来调试Node.js, 你会感到很困扰, 大部分新加入node的开发者采用了’flow’的调试模式,从这一刻开始你最好的朋友就是console.log

Node-inspector 可以让你做一些非常酷的事情,比如实时修改,单步调试,注入以及一堆其他非常酷的东西.

贴士 5: 利用NPM进行模块管理

Node做最常见的事情之一是通过NPM安装软件包。Node有一个惊人的包管理器安装所有指定在你的package.json的manifest文件中的模块。然而，所有初学者碰上保持的package.json文件更新您所使用的所有的模块。

这似乎是一个痛苦的总是打开的package.json来更新新模块的依赖，但许多人不知道的是npm会为你做这个！

非常简单运行 npm install - save module_name 然后 npm将自动更新你的package.json 包含正确的模块和版本,

贴士 6: 不要检查node_modules 文件夹

虽然我们的话题模块和npm，不是很多人都知道，你不应该检查node_modules文件夹中。这背后最大的原因是，有没有必要为你检查这个文件夹中。只要有人检查你的源出，他们可以只运行NPM安装和下载所有需要的模块。

您可能会说，它是不是一个大问题，如果检查node_modules，但是，如果签出的人使用了和你编译不一样的操作系统的来安装通过NPM？你的应用程序将会崩溃，签出人将不知道为什么！

举个例子 bcrypt 以及 sentimental，如当在您安装在主机系统上编译他们，因为他们有C语言编写的本地组件.

避免检查node_modules文件夹的方式是加入.gitignore

// .gitignore node_modules/*

Octopress Can not create new post on zsh

Aug 10th, 2013 Comments

执行：$ rake new_post[“arch-linux-reinstall-glibc.markdown”]

报错：zsh: no matches found: new_post[arch-linux-reinstall-glibc]

原因：zsh中若出现‘*’, ‘(’, ‘|’, ‘<’, ‘[’, or ‘?’符号，则将其识别为查找文件名的通配符

快速解决：用引号括起来$ rake “new_post[arch-linux-reinstall-glibc.markdown]”

彻底解决：取消zsh的通配(GLOB), 在.zshrc中加入alias rake=”noglob rake”
